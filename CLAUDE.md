# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

Terraform infrastructure for AI South Hub 2026 hackathon on Cloud.ru Evolution. Single subnet with edge/NAT VM (bastion + Traefik + Xray TPROXY, `interface_security_enabled=false`) and team VMs — all on 10.0.1.0/24. Post-deploy configuration via Ansible. All documentation is in Russian.

Domain: `south.aitalenthub.ru` — teams get `{team_id}.south.aitalenthub.ru` via edge Traefik.

## Key Commands

```bash
# All terraform commands run from the environment directory
cd environments/dev

# Format all .tf files (required before commit)
terraform fmt -recursive

# Validate configuration
terraform init
terraform validate

# Plan and apply
terraform plan
terraform apply

# Full format check (CI-style)
terraform fmt -check -recursive

# Ansible (after terraform apply)
cd ansible

# PREREQUISITE: secrets/admin-keys.txt must exist with at least one SSH public key
# echo "ssh-ed25519 AAAA... admin@example.com" > ../../secrets/admin-keys.txt

ansible-playbook playbooks/edge.yml       # edge VM: NAT, Traefik (HTTPS+ACME), Xray
ansible-playbook playbooks/team-vms.yml   # team VMs: route override, Docker, team-traefik

# Sync SSH keys only (without reconfiguring services)
ansible-playbook playbooks/sync-keys.yml

# Update services without full reconfiguration
ansible-playbook playbooks/update-services.yml
```

## Architecture

**Single-subnet VM-router topology (10.0.1.0/24):**
- **Edge VM** (10.0.1.10): floating IP, `interface_security_enabled=false` (no SG — firewall via iptables). Runs:
  - **Traefik** (Docker, `network_mode: host`) — HTTPS termination with ACME HTTP-01, routes `{team_id}.south.aitalenthub.ru` → team VMs; dynamic config in `/etc/traefik/dynamic/{team_id}.yml`
  - **Xray** (systemd) — transparent AI API proxying via TPROXY
  - **iptables** NAT/MASQUERADE + hairpin NAT for team VMs
- **Team VMs** (10.0.1.x): same subnet, SG restricts ingress to edge IP. Default route overridden to edge via netplan (Ansible). Runs:
  - **team-traefik** (Docker) — HTTP-only, docker provider, port 80; exposes team's services to edge Traefik

**Infrastructure split:**
- **Terraform**: VM provisioning, networking, security group (team only), SSH key generation
- **Ansible**: all post-deploy configuration (Docker, Traefik, Xray, NAT/iptables rules, route override, services)

**Module dependency chain:**
```
data sources (AZ, flavor, disk_type)
  → network → security → edge  → team_vm
                                → team-credentials
                                → ansible inventory
```

## Module Layout

Each module under `modules/` follows: `main.tf`, `variables.tf`, `outputs.tf`, `versions.tf`. The entry point environment is `environments/dev/`.

| Module | Purpose |
|--------|---------|
| `network` | Single subnet (10.0.1.0/24), `routed_network=true` in VPC Default |
| `security` | Security group for team VMs only (inline rules, edge IP /32 + subnet CIDR) |
| `edge` | Edge/NAT VM with floating IP, single interface, `interface_security_enabled=false` |
| `team_vm` | Per-team VMs with static IPs (10.0.1.x) |
| `team-credentials` | SSH key generation and credential file management |

## Ansible Structure

```
ansible/
├── playbooks/     # edge.yml, team-vms.yml, sync-keys.yml, update-services.yml
├── roles/         # common, docker, nat, traefik, team-traefik, xray
├── group_vars/    # all.yml, edge.yml, team_vms.yml
├── inventory/     # hosts.yml (generated by Terraform)
└── templates/     # inventory.yml.tpl (Terraform templatefile)
```

| Role | Runs on | Purpose |
|------|---------|---------|
| `common` | all | base packages, Claude Code (teams only, skipped on edge), Playwright (teams only) |
| `docker` | all | Docker Engine install |
| `nat` | edge | iptables NAT/MASQUERADE + hairpin, iptables-persistent |
| `traefik` | edge | Traefik with HTTPS/ACME, per-team dynamic routing |
| `team-traefik` | team VMs | HTTP-only Traefik with docker provider |
| `xray` | edge | Xray TPROXY for AI API access |

## Important Patterns

- **Provider**: Cloud.ru Evolution (`cloud.ru/cloudru/cloud` v1.6.0), installed via filesystem mirror
- **Auth**: `project_id`, `auth_key_id`, `auth_secret` — passed via `terraform.tfvars` (gitignored)
- **Data sources**: AZ, flavor, disk type looked up dynamically in `environments/dev/main.tf`
- **SSH keys**: **1 key per team** (`tls_private_key.team_key[team_id]`, ED25519). Used for both bastion jump (with `permitopen` restriction) and VM login. Previously was 3 keys — `moved {}` blocks handle migration.
- **Static IPs**: edge=10.0.1.10, team VMs get IPs via `ip` field in teams map (e.g., dashboard=10.0.1.100)
- **Teams map**: defined in `terraform.tfvars` as a map keyed by team ID (e.g., `"dashboard"`) with `user`, `public_keys`, and `ip` fields
- **Ansible inventory**: auto-generated by Terraform from `ansible/templates/inventory.yml.tpl`
- **secrets/admin-keys.txt**: one SSH public key per line, comments (`#`) and blank lines ignored. Required by `sync-keys.yml` — without it playbook fails. Not generated by Terraform, must be created manually.
- **Participant credentials**: `team-credentials` module generates per-team folder in `secrets/team-{id}/` with: `{id}-key`, `{id}-key.pub`, `ssh-config`, `setup.sh`, `setup.bat` (CMD), `setup.ps1` (UTF-8 BOM for PS5), `README.md`. Plus `secrets/teams-credentials.json` summary.

## Cloud.ru Evolution Specifics

- No VPC resource — subnets are top-level resources
- No route table resource — static routes configured via Ansible on team VMs
- Security group rules are inline (`rules {}` blocks), source is CIDR (`remote_ip_prefix`)
- All resources require `availability_zone` block
- Subnet names must be >= 7 characters
- Port ranges format: `"from:to"` or `"any"`
- Compute `image` block sets hostname, username, SSH key (no cloud-init/user_data support)

## Coding Standards

- `terraform fmt` before every commit
- Resources use snake_case: `cloudru_evolution_compute.edge`
- Modules use kebab-case directory names: `team-credentials`
- Comments explain "why", not "what"
- Commit messages follow conventional commits: `feat(edge):`, `fix(xray):`, `docs(quickstart):`
- Branch naming: `feature/`, `fix/`, `docs/`, `refactor/`

## Secrets (gitignored)

The `secrets/` directory contains generated SSH keys and `xray-config.json`. All are gitignored except `.gitkeep`. Never commit these files.
